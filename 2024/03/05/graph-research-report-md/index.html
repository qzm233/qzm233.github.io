GNN in traffic

TraffNet: Learning Causality of Traffic Generation for What-if Prediction

The network is shown below.

The network basically contains several components:

  • Trip Graph Construction

    The data contains:

    • Segments: $v_1 \dots v_n$ represents the road segments, seen as nodes, linked by intersections.
    • OD: <$v_i,v_j, t$> Origin-destination, the data record the number of vehicles travel from $v_i$ to $v_j$ at time interval $t$
    • Path <$v_1 \dots v_{|p|},t$> record the number of vehicles travel via certain path at time $t$

  • Path embedding Initialization

    The intuition is to encode the path by regarding paths as sequences just like natural language and use a bi-direction RNN-like net.

    One will get the embedding of each segment and thus get the path embedding by concatenation of segment embedding. The results are called $\hat H_i$

  • TAL layer

    • the Route Learning

      • Determine the proportion of OD demands assigned to each candidate route under dynamic traffic conditions

      • Specifically, it uses the path data and Graph Attention to learn the importance of a path-node and its neighbors of the same OD.

      • From my comprehension, the network calculate the attention scores as weights and update the path embedding

      (needs more explanation)

    • Segment Message Passing

      • Update segment embedding according to the path-node embedding

      (needs more explanation)

  • Temporal Dependencies Modeling

    • Predict future traffic volumes for each road segment based on observed OD demands and traffic conditions. Use a RNN-like decoder.

    • Feed all the segment embeddings $\hat H_t$ as a whole from the previous $t$ and predict traffic volume

Graph Anomaly Detection

Task discripthion:

Anomaly detection aims to identify data points or patterns that significantly deviate from the norm within graph-structured data, particularly when the graph nodes are associated with attributes.

In attributed graphs, each node typically has associated attributes (e.g., in a social network, users might have attributes like age, gender, interests), which provide rich information for analysis. Anomaly detection is especially important for such data as it can help identify and prevent various abnormal behaviors, such as fraud, intrusions, disease outbreaks, etc.
  • Cross-Domain Graph Anomaly Detection
    The network basically contains 4 components

    • A graph attentive encoder

      Based on GATs.

    • A domain discriminator

        After the feature encoding phase, the domain label of nodes would not be accurately recognized by the domain discriminator, and the shared graph attentive encoder would be able to extract domain-invariant node representations from both source graph $G_s$ and target graph $G_t$

    • An anomaly classifier

    • An attribute decoder.

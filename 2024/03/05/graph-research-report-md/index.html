TraffNet: Learning Causality of Traffic Generation for What-if Prediction

The network is shown below. The network basically contains several components:

  • Trip Graph Construction

    The data contains:

    • Segments: represents the road segments, seen as nodes, linked by intersections.
    • OD: <> Origin-destination, the data record the number of vehicles travel from to at time interval
    • Path <> record the number of vehicles travel via certain path at time

  • Path embedding Initialization

    • The intuition is to encode the path by regarding paths as sequences just like natural language and use a bi-direction RNN-like net.
    • One will get the embedding of each segment and thus get the path embedding by concatenation of segment embedding. The results are called

  • TAL layer

    • the Route Learning

      • Determine the proportion of OD demands assigned to each candidate route under dynamic traffic conditions
      • Specifically, it uses the path data and Graph Attention to learn the importance of a path-node and its neighbors of the same OD.

Graph Neural Networks for Road Safety Modeling: Datasets and Evaluations for Accident Analysis

  • Objective: Analyze traffic accidents on road networks using historical records
  • Task: Given a period traffic accident records, pridict the remaining period:
    • of whether an accicent occur or not(as a classification task)
    • of the number of accidents per edge, say road(as a regression task)
  • Methodology:
    • Dataset: Accident data(within an edge), Annual average daily traffic reports, Road features(both intersections and roads)
    • GNN model for accident prediction:
      • GNN for node(intersection) embedding(from feature of both nodes and edges)
      • Cross-state analysis: Multitask learning models to capture cross-state variability. Use the shared encoder for all states and design a separate prediction layer. Minimize the average loss for all tasks.
      • Combining road network geometry and traffic flow. Shared encoder and separate prediction layers for two tasks: one for primary accident prediction and another for auxiliary volume prediction.
  • Experiments:
    • Phase 1: Several graph embedding methods based on just nodes and edges features
    • Phase 2.1: Multitask Learning(MTL) on combned data of eight states (more imformation about label is needed)
    • Phase 2.2: MTL finetuning(MTL-FT) for each state(more imformation about label is needed)
    • Phase 2.3: Transfer learning(TL) on accident and volume prediction jointly.

Graph Anomaly Detection

Task discripthion:

  • Anomaly detection aims to identify data points or patterns that significantly deviate from the norm within graph-structured data, particularly when the graph nodes are associated with attributes.
  • In attributed graphs, each node typically has associated attributes (e.g., in a social network, users might have attributes like age, gender, interests), which provide rich information for analysis. Anomaly detection is especially important for such data as it can help identify and prevent various abnormal behaviors, such as fraud, intrusions, disease outbreaks, etc.
  • Cross-Domain Graph Anomaly Detection The network basically contains 4 components

    • A graph attentive encoder

      Based on GATs.

    • A domain discriminator

      • After the feature encoding phase, the domain label of nodes would not be accurately recognized by the domain discriminator, and the shared graph attentive encoder would be able to extract domain-invariant node representations from both source graph and target graph

    • An anomaly classifier

    • An attribute decoder.
